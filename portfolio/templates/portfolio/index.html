Selected RAG implementations demonstrating scale and accuracy.

Offline Ingestion Pipeline

A highly parallelized ingestion pipeline built with Python and Celery. It processes thousands of internal PDFs and Confluence pages daily. Implements semantic chunking and layout-aware parsing before embedding via OpenAI's text-embedding-3-large into a Qdrant cluster.

❯ Architecture highlight: Metadata-enriched chunking
❯ Scale: 5M+ vectors, >95% retrieval accuracy@K=5

Online Streaming Orchestrator

Low-latency RAG implementation for customer-facing chatbots. Intercepts queries, applies query expansion via a small LM, and retrieves context from Redis vector hashes in under 150ms. Streams LLM responses back via WebSockets.

❯ Architecture highlight: Query rewriting & routing
❯ Latency: P99 < 800ms TTFT (Time To First Token)

Advanced Re-ranking Engine

Solves exactly-matching issues in standard vector search by combining BM25 keyword search with dense embeddings. A second-stage cross-encoder re-ranks the top 50 results to ensure the most factually relevant context is passed to the generation model.

❯ Architecture highlight: Two-stage retrieval + Cohere Re-rank
❯ Result: 40% reduction in LLM hallucinations
